汇总当前对话中的知识要点，保存为笔记（支持新建或追加到已有笔记）。

## 用法

/note
/note append

## 行为

1. AI 回顾当前对话中最近的问答内容（特别是 AI 解答的知识性内容）
2. 汇总为笔记：
   - 自动生成标题（简明概括主题）
   - 除非重复性的内容，否则不要删减AI的解答
   - 正文为 Markdown 格式，结构清晰，包含关键要点、代码示例等
   - **正文开头必须插入时间戳行**：`> 📅 记录于 YYYY-MM-DD HH:MM`（使用当前实际时间）
3. 展示汇总预览给用户确认
4. 判断模式：
   - 如果用户传了 `append` 参数，或在确认时选择「追加到已有笔记」→ 进入追加流程
   - 否则 → 进入新建流程

### 新建流程

5a. 标签智能推荐：
    - 调用 `list_notes(limit=50)` 获取历史笔记，提取所有已有标签（去重）
    - AI 根据笔记内容判断是否匹配已有标签：
      - 若匹配 → 将匹配的标签排在候选列表前面
      - 若不匹配任何已有标签 → AI 自动生成一个合适的新标签名称，加入候选列表末尾并标注「🆕 新标签」
    - 展示候选标签列表给用户选择（支持多选）
6a. 调用 MCP `create_note(title, content, note_type="速记", tags)` 保存
7a. 输出确认：

📝 笔记已保存「{标题}」
   标签: {tag1}, {tag2}
   🔗 {notion_url}

### 追加流程

5b. 调用 `search_notes(query)` 用汇总标题关键词搜索相关笔记
    - 如果无结果，调用 `list_notes(limit=10)` 展示最近笔记
6b. 列出候选笔记，让用户选择目标笔记
7b. 调用 MCP `append_note(note_id, content)` 追加内容（内容开头插入 `> 📅 追加于 YYYY-MM-DD HH:MM`）
8b. 输出确认：

📝 已追加到「{笔记标题}」
   🔗 {notion_url}
