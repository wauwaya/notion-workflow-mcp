查看所有未完成任务，按项目分组展示，包含子目标。

## 用法

/task_list [-p 项目] [-s 状态]

## 示例

/task_list
/task_list -p MCP改造
/task_list -s 进行中

## 行为

1. 调用 MCP `list_tasks(project, status)` 获取任务列表
   - 默认排除状态为"完成"的任务（除非 `-s 完成`）
   - 如果未指定 `-s`，查询"待办"和"进行中"两次并合并
2. 对每个任务调用 MCP `get_subtasks(task_id)` 获取子目标
   - 注意：任务较多时可能耗时数秒，应提示用户"加载中..."
3. 按项目分组，渲染输出：

📋 未完成任务

### 项目: MCP改造
1. 重构Notion MCP [{task_id前8位}] (🟡 高, 截止3/28)
   - [ ] 去掉关联功能 (🟡 高)
   - [~] 增加项目字段 (🔴 紧急)
   - [ ] 创建skill (🟢 普通)

### 项目: 业务开发
2. 用户模块优化 [{task_id前8位}] (🟢 普通)
   - [x] 修复登录bug (🟡 高)
   - [ ] 优化查询性能 (🟢 普通)

### 无项目
3. 其他任务 [{task_id前8位}] (🟢 普通)

4. 无项目的任务归入"无项目"组，排在最后
